# Jarvis Voice Sight v0.5 Completion Audit

Recorded date: 2026-06-10

## Result

v0.5 realtime voice agent prototype is implemented and verified for the local
Ollama/Breeze-ASR/BreezyVoice demo path. vLLM is implemented as a configurable
OpenAI-compatible LLM runtime path; live vLLM smoke requires an external vLLM
server at `VLLM_BASE_URL`.

## Requirement Evidence

| Requirement | Evidence |
| --- | --- |
| ASR / LLM / TTS provider abstraction | `services/orchestrator/src/ports/modelPorts.ts` defines `ASRPort`, `LLMPort`, `TTSPort` plus v0.5 aliases `AsrProvider`, `LlmProvider`, `TtsProvider`. |
| Ollama / vLLM configurable LLM runtime | `services/orchestrator/src/config/env.ts`, `services/orchestrator/src/factories/AdapterFactory.ts`, and `services/llm/src/server.py` support `LLM_PROVIDER=ollama|vllm`, `LLM_RUNTIME=ollama|vllm`, `OLLAMA_*`, and `VLLM_*`. Unit tests cover provider switching. |
| VAD state manager and always-listening mode | `packages/shared/src/types/realtime.ts`, `apps/web/src/lib/realtime.ts`, and `apps/web/src/hooks/useRealtimeVoiceController.ts` implement the required states and thresholds. Unit tests cover listening to ASR transition. |
| Barge-in support | `useRealtimeVoiceController` calls `audio.stop()` and `voiceTurn.cancelTurn()` on interruption; unit and smoke tests verify `interrupted_then_listening`. |
| Turn isolation and stale audio discard | `TurnPlaybackGuard`, `TtsQueue.cancel`, `useVoiceTurn`, and `useAudioPlayback` guard playback by active `turn_id`. Tests cover stale playback guard and no duplicate playback across 50 turns. |
| Natural Taiwanese Mandarin persona | `services/orchestrator/src/strategies/ResponseStrategy.ts`, `services/llm/src/server.py`, response policy, repair, and canonicalizer keep short Traditional Chinese replies and avoid always asking questions. |
| Response finalizer before TTS | `services/orchestrator/src/policy/TtsTextFinalizer.ts` removes markdown, JSON, role tags, URLs, emojis, and excess length before TTS. Unit tests cover finalizer behavior. |
| Sentence-level LLM streaming to TTS queue | `StreamingVoiceTurnUseCase.streamReplyToTts`, `SentenceBuffer`, and `/api/v1/voice-turn-stream` stream tokens into complete sentence chunks. Integration tests cover sentence-to-TTS chunk flow. |
| Parallel TTS chunk synthesis | `StreamingVoiceTurnUseCase.streamLongFormReplyToTts` plans chunks, starts bounded concurrent syntheses, and yields ordered audio chunks. Integration tests verify ordered chunks, first-audio-before-all-complete, cache reuse, and abort behavior. |
| WAV normalize, silence padding, audio stitching contract | `buildMergedAudioMetadata` and streaming completion `audio_stitch` expose sample-rate verification, normalization, silence padding, and total duration metadata. Integration tests assert this contract. |
| Deterministic TTS cache | `services/tts/src/server.py` caches normalized text to deterministic wav paths. TTS cache tests cover hit, miss, normalization, warmup, emotion-style reuse, and `<500ms` cache hit. |
| Latency metrics | `LatencyReport` and voice-turn completion include `vad_ms`, `asr_ms`, `llm_first_token_ms`, `llm_total_ms`, `tts_first_audio_ms`, `tts_total_ms`, `playback_start_ms`, `total_ms`, `tts_cache_hit`, and `tts_parallel_chunks` while preserving compatibility fields. Live `/api/v1/voice-turn` contract check passed. |
| UI runtime state display | `apps/web/src/lib/constants.ts` and `apps/web/src/app/page.tsx` visibly display Listening, Understanding, Thinking, Speaking, Interrupted, and Error recovery. |
| `npm run demo:real` | `scripts/demo_real.sh` starts or reuses real services, checks Ollama or vLLM by env, runs health/preflight, and prints `Ready for Demo`. Live command passed. |
| Required tests | Unit/integration/Python tests cover provider switching, VAD transitions, barge-in, turn cancellation, stale audio discard, cache hit, cache miss, sentence chunking, parallel TTS, latency output, and no duplicate playback across 50 turns. |

## Operating Note

The local demo path is ready at:

```text
Web UI: http://localhost:3001
Orchestrator: http://localhost:3000
```

To switch to vLLM, run an OpenAI-compatible vLLM server first, then set:

```env
LLM_PROVIDER=vllm
LLM_RUNTIME=vllm
VLLM_BASE_URL=http://localhost:8000/v1
VLLM_MODEL=gemma-4-e2b
```
